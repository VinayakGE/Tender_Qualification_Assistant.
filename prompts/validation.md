# Prompt: JSON Validator

**Version:** 1.0.0  
**Stage:** Validation (cross-cutting)  
**Model:** claude-sonnet-4-6  
**Last updated:** 2024-06-01

---

## Purpose

Validate LLM-generated JSON output against expected schemas and flag errors with actionable correction instructions. Used as a fallback when Pydantic validation fails and a retry is warranted.

---

## Prompt

```
You are a JSON data validator. You will receive a JSON object or array that was generated by a language model, and a description of what it should contain.

Your task is to:
1. Check if the JSON is structurally valid
2. Identify any violations of the rules below
3. Return a corrected version

VALIDATION RULES:
- All required fields must be present and non-null (unless the field explicitly allows null)
- Enum fields must contain only the allowed values
- Numeric fields must contain numbers, not strings (e.g., 30 not "30")
- Boolean fields must contain true or false, not "true" or "false" or 1 or 0
- Array fields must be arrays, even if there is only one item
- String fields must be strings, not null unless null is explicitly allowed
- Date fields must be in ISO 8601 format (YYYY-MM-DD for dates, YYYY-MM-DDTHH:MM:SSZ for datetimes)

EXPECTED SCHEMA:
{schema_description}

INPUT JSON:
{input_json}

OUTPUT:
Return a JSON object with exactly two fields:
{
  "valid": true | false,
  "corrected_json": <the corrected JSON, or the original if no corrections were needed>,
  "errors_found": ["list of errors found and corrected, or empty array if none"]
}

If the input is so malformed that it cannot be corrected (e.g., not valid JSON at all, or missing all required fields), set valid to false, corrected_json to null, and list the errors found.

IMPORTANT: Return ONLY the validation result JSON. No preamble.
```

## Usage in Code

```python
# In src/extractor/requirement_extractor.py
from src.utils.helpers import load_json

def _validate_and_retry(self, raw_response: str, schema_description: str) -> list[dict]:
    """Validate LLM response; retry with correction if invalid."""
    prompt = self._load_validation_prompt(raw_response, schema_description)
    result = self._call_claude(prompt)
    validation_result = json.loads(result)
    
    if validation_result["valid"]:
        return json.loads(raw_response)
    
    if validation_result["corrected_json"]:
        logger.warning("requirement_extraction_corrected", 
                       errors=validation_result["errors_found"])
        return validation_result["corrected_json"]
    
    raise ValueError(f"Could not parse requirement JSON: {validation_result['errors_found']}")
```

## Notes for Maintainers

- This prompt is a defensive layer, not the primary validation mechanism. Pydantic validation runs first.
- The `corrected_json` approach means the system is self-healing for common LLM output errors (string numbers, missing nulls, wrong enum values).
- Do not use this prompt for business logic validation (e.g., "is this threshold reasonable?"). It is for structural JSON validation only.
- If the corrected JSON also fails Pydantic validation after this step, log the error and skip the problematic item — do not retry indefinitely.
